Build OpenCNAPP
The Open-Source Way

A self-hosted, multi-cloud, fully modular security platform. Runs locally via Docker Compose. Supports Azure, AWS, GCP, with or without Kubernetes, with or without CI/CD — every component optional and pluggable.

Local Docker Compose Multi-Cloud No K8s required Plugin Architecture 100% Open Source
01

Design philosophy — built for everyone

The core shift from the previous plan: nothing is assumed. Every tool, every cloud, every integration is optional. Users adopt what they have.

What changed from v1
  • No Azure/AKS assumption — runs on your laptop or a $5/mo VM
  • No Kubernetes required — optional K8s plugin only
  • No CI/CD assumption — tools work standalone OR integrated
  • Every scanner is a plugin — enable what you have, skip what you don't
  • Multi-cloud from day 1 — Azure, AWS, GCP as equal first-class connectors
Three user archetypes we support
Solo dev  Running locally, scanning one AWS account, no K8s, basic CI/CD via GitHub Actions
SMB team  Self-hosted VM, Azure + some K8s, mixed CI/CD (GitHub + Azure DevOps), Slack alerts
Growing company  Multi-cloud, AKS+EKS, full CI/CD gates, custom pentest schedules, public dashboard
The golden rule: The platform should work with zero plugins enabled (just the UI + database). Each plugin you enable adds coverage. The more you enable, the more it looks like Wiz. But none are required to start.

Local-first architecture — runs on Docker Compose

The entire platform runs as Docker Compose on any Linux/Mac/Windows machine. No cloud required for the platform itself. Cloud credentials are injected via environment variables to scanners.

Deployment model: Platform runs locally (Docker Compose) or on a single VM. Scanners reach out to clouds via APIs — they don't need to run in the cloud. The dashboard is a web UI accessed via browser. Everything talks over a private Docker network.
Option A — Developer laptop
docker compose up on Mac/Linux/Windows WSL2. Scans your personal cloud accounts. Dashboard at localhost:3000. Ideal for solo security engineers.
Option B — Internal VM
Deploy on a $20/mo Ubuntu VM. Internal team access. All company cloud accounts connected. Persistent storage. Recommended for teams of 2-10.
Option C — Self-hosted server
On-prem server or company-owned cloud VM. Nginx + TLS in front. LDAP/SSO auth. Persistent volumes. For larger teams wanting full control.

Plugin system — the core of flexibility

Every scanner is a plugin. Each plugin is self-contained: Docker image + normalizer adapter + default schedule + required config. Users enable only what they have.

Plugin manifest — each scanner defines itself (YAML)YAML
# plugins/prowler/plugin.yaml
id: prowler
name: Prowler CSPM
domain: cspm
description: 500+ security checks for AWS, Azure, GCP
version: "4.x"

# Which clouds this plugin supports
clouds_supported: [aws, azure, gcp]
requires_k8s: false
requires_cicd: false

# Docker image to run
image: toniblyx/prowler:latest

# Config inputs — shown in dashboard Plugin Manager UI
config_schema:
  - key: CLOUD_PROVIDER
    label: Cloud provider
    type: select
    options: [aws, azure, gcp]
    required: true
  - key: AZURE_SUBSCRIPTION_ID
    label: Azure subscription ID
    type: string
    required_if: "CLOUD_PROVIDER == azure"
  - key: AWS_PROFILE
    label: AWS profile name
    type: string
    required_if: "CLOUD_PROVIDER == aws"

# Default schedule — user can override in UI
default_schedule: "0 */6 * * *"   # every 6 hours
run_mode: scheduled                # scheduled | on_demand | continuous

# Command template — vars filled from config
command:
  - prowler
  - "{{ CLOUD_PROVIDER }}"
  - -M json
  - -o /output/results.json

output:
  format: json
  path: /output/results.json
  normalizer: prowler_normalizer   # maps to normalizers/prowler.py

# UI features this plugin contributes
dashboard_widgets:
  - type: finding_count_by_check
  - type: severity_breakdown

tags: [cspm, azure, aws, gcp, cis-benchmark]
Adding a new tool is just: (1) write a plugin.yaml, (2) write a normalizer adapter (~50 lines of Python), (3) drop both in the plugins/ folder. The platform auto-discovers it. No code changes to the core platform needed.

Universal findings schema

PostgreSQL schema — cloud-agnostic, tool-agnosticSQL
CREATE TABLE plugins (
  id           TEXT PRIMARY KEY,             -- e.g. 'prowler', 'kubescape'
  name         TEXT, domain TEXT, version TEXT,
  enabled      BOOLEAN DEFAULT FALSE,
  config       JSONB,                           -- user-provided credentials/settings
  schedule     TEXT,                            -- cron expression
  run_mode     TEXT                             -- scheduled|on_demand|continuous
);

CREATE TABLE connectors (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  cloud        TEXT,                            -- aws | azure | gcp | onprem | saas
  name         TEXT,                            -- user-given name e.g. "prod-azure"
  credentials  JSONB,                           -- encrypted at rest
  metadata     JSONB,                           -- subscription_id, account_id, etc.
  created_at   TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE scans (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  plugin_id     TEXT REFERENCES plugins(id),
  connector_id  UUID REFERENCES connectors(id),
  trigger       TEXT,                           -- scheduled | manual | cicd | webhook
  status        TEXT,                           -- queued|running|completed|failed
  started_at    TIMESTAMPTZ,
  finished_at   TIMESTAMPTZ,
  summary       JSONB                           -- {critical:3, high:12, medium:45}
);

CREATE TABLE findings (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  scan_id       UUID REFERENCES scans(id),
  plugin_id     TEXT,
  domain        TEXT,                           -- cspm|kspm|cwpp|ciem|image|iac|secrets|sspm|pentest
  cloud         TEXT,                           -- aws|azure|gcp|onprem|none
  severity      TEXT,                           -- CRITICAL|HIGH|MEDIUM|LOW|INFO
  resource_type TEXT,
  resource_id   TEXT,
  resource_name TEXT,
  region        TEXT,
  check_id      TEXT,
  title         TEXT NOT NULL,
  description   TEXT,
  remediation   TEXT,
  references    JSONB,                          -- [{url, title}]
  frameworks    TEXT[],                         -- ['cis-1.4', 'pci-dss-3.2', 'soc2']
  raw_output    JSONB,
  status        TEXT DEFAULT 'open',          -- open|acknowledged|suppressed|resolved
  assigned_to   TEXT,
  first_seen    TIMESTAMPTZ DEFAULT NOW(),
  last_seen     TIMESTAMPTZ DEFAULT NOW(),
  tags          TEXT[] DEFAULT '{}'
);

Three data flows

Flow 1 — Scheduled scan
  1. APScheduler fires cron trigger
  2. Pulls plugin config + connector creds
  3. Runs Docker container for scanner
  4. Container outputs JSON to shared volume
  5. Normalizer maps → common schema
  6. Upsert to PostgreSQL findings table
  7. Dashboard refreshes on next poll
Flow 2 — UI-triggered pentest
  1. User clicks "Run Nuclei" in dashboard
  2. POST /api/trigger-scan { plugin, params }
  3. FastAPI → Redis job queue
  4. Celery worker picks up job
  5. Runs scanner Docker container
  6. Normalizer → PostgreSQL
  7. WebSocket pushes "complete" + results to UI
Flow 3 — CI/CD pipeline
  1. PR opened / commit pushed
  2. Pipeline runs Trivy, Checkov, Gitleaks
  3. Each tool outputs JSON artifact
  4. POST to /api/ingest with tool name + JSON
  5. Normalizer maps → findings
  6. PR gets comment with finding count
  7. Gate fails if CRITICAL findings exist
Falco is different: it runs as a daemon (not triggered) and pushes alerts via webhook to POST /webhook/falco in real time. FastAPI writes them as CRITICAL findings and broadcasts to all WebSocket clients immediately.

CSPM — Prowler setup

docker-compose profile for Prowler — multi-cloudyaml
# In docker-compose.yml — prowler service
prowler-scheduler:
  image: toniblyx/prowler:latest
  profiles: [prowler, all]
  volumes:
    - ./output/prowler:/output
    - ./plugins/prowler:/plugin:ro
  environment:
    # Azure
    AZURE_CLIENT_ID:       ${AZURE_CLIENT_ID:-}
    AZURE_CLIENT_SECRET:   ${AZURE_CLIENT_SECRET:-}
    AZURE_TENANT_ID:       ${AZURE_TENANT_ID:-}
    AZURE_SUBSCRIPTION_ID: ${AZURE_SUBSCRIPTION_ID:-}
    # AWS
    AWS_ACCESS_KEY_ID:     ${AWS_ACCESS_KEY_ID:-}
    AWS_SECRET_ACCESS_KEY: ${AWS_SECRET_ACCESS_KEY:-}
    AWS_DEFAULT_REGION:    ${AWS_DEFAULT_REGION:-us-east-1}
    # Normalizer config
    DATABASE_URL:          postgresql://cnapp:${POSTGRES_PASSWORD}@postgres/cnapp
    SCAN_TOOL:             prowler
  networks: [cnapp-net]
  # Not long-running — triggered by scheduler service
  restart: "no"

The scheduler service calls docker compose run prowler-scheduler prowler azure -M json -o /output/results.json on the configured cron. Output lands in ./output/prowler/, normalizer picks it up.

CI/CD integrations — push findings to OpenCNAPP

CI/CD tools run in the pipeline and POST results to your OpenCNAPP instance. Works with any CI/CD system. No CI/CD at all? Just use the scheduled scanner mode instead.

GitHub Actions — Trivy + Checkov + Gitleaks → OpenCNAPPyaml
# .github/workflows/security-scan.yml
name: Security Scan
on: [push, pull_request]

jobs:
  security:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }     # full history for gitleaks

      # 1. Gitleaks — secrets in this commit/PR
      - name: Gitleaks
        uses: gitleaks/gitleaks-action@v2
        continue-on-error: true
        id: gitleaks
        env: { GITHUB_TOKEN: "${{ secrets.GITHUB_TOKEN }}" }

      # 2. Trivy — IaC + image scanning
      - name: Trivy IaC scan
        uses: aquasecurity/trivy-action@master
        with:
          scan-type: config
          format: json
          output: trivy-iac.json
          exit-code: '0'            # don't fail pipeline here, let CNAPP decide

      # 3. Checkov — deeper IaC
      - name: Checkov
        uses: bridgecrewio/checkov-action@master
        with:
          output_format: json
          output_file_path: checkov.json
          soft_fail: true

      # 4. POST all results to your OpenCNAPP instance
      - name: Send to OpenCNAPP
        run: |
          for tool in gitleaks trivy-iac checkov; do
            [ -f "${tool}.json" ] && curl -s -X POST \
              "${{ secrets.OPENCNAPP_URL }}/api/ingest" \
              -H "Authorization: Bearer ${{ secrets.OPENCNAPP_TOKEN }}" \
              -H "Content-Type: application/json" \
              -d "{\"tool\": \"${tool}\", \"trigger\": \"cicd\",
                   \"repo\": \"${{ github.repository }}\",
                   \"commit\": \"${{ github.sha }}\",
                   \"data\": $(cat ${tool}.json)}"
          done

      # 5. Optionally: comment on PR with finding summary
      - name: PR comment
        if: github.event_name == 'pull_request'
        run: |
          SUMMARY=$(curl -s "${{ secrets.OPENCNAPP_URL }}/api/scan-summary/latest" \
            -H "Authorization: Bearer ${{ secrets.OPENCNAPP_TOKEN }}")
          gh pr comment ${{ github.event.number }} --body "$SUMMARY"
        env: { GH_TOKEN: "${{ secrets.GITHUB_TOKEN }}" }

# workers/pentest_runner.py
import docker, uuid, json
from celery import Celery
from .normalizers import get_normalizer
from .database import insert_findings, update_scan_status
from .websocket import broadcast

app = Celery("cnapp", broker="redis://redis:6379/0")
client = docker.from_env()

PENTEST_TOOLS = {
    "nuclei": {
        "image": "projectdiscovery/nuclei:latest",
        "cmd":   lambda p: ["nuclei","-u",p["target"],"-json","-o","/output/out.json"]
    },
    "kube-hunter": {
        "image": "aquasec/kube-hunter:latest",
        "cmd":   lambda p: ["kube-hunter","--pod","--report","json"]
    },
    "nmap": {
        "image": "instrumentisto/nmap:latest",
        "cmd":   lambda p: ["nmap","-sV","-oJ","/output/out.json",p["target"]]
    },
    "sslyze": {
        "image": "nablac0d3/sslyze:latest",
        "cmd":   lambda p: ["sslyze","--json_out=/output/out.json",p["target"]]
    },
    "nikto": {
        "image": "frapsoft/nikto:latest",
        "cmd":   lambda p: ["nikto","-h",p["target"],"-Format","json","-output","/output/out.json"]
    },
}

@app.task
def run_pentest(tool: str, scan_id: str, params: dict):
    update_scan_status(scan_id, "running")
    output_dir = f"/tmp/cnapp-scans/{scan_id}"

    try:
        cfg = PENTEST_TOOLS[tool]
        container = client.containers.run(
            cfg["image"],
            command=cfg["cmd"](params),
            volumes={output_dir: {"bind": "/output", "mode": "rw"}},
            remove=True, detach=False
        )
        with open(f"{output_dir}/out.json") as f:
            raw = json.load(f)

        normalizer = get_normalizer(tool)
        findings = normalizer.normalize(raw, scan_id)
        insert_findings(findings)
        update_scan_status(scan_id, "completed", len(findings))
        broadcast({"event": "scan_complete", "scan_id": scan_id, "count": len(findings)})

    except Exception as e:
        update_scan_status(scan_id, "failed", error=str(e))
        broadcast({"event": "scan_failed", "scan_id": scan_id, "error": str(e)})

Base normalizer class + Prowler adapterpython
# normalizers/base.py
from abc import ABC, abstractmethod
from typing import List
import uuid
from datetime import datetime, timezone

SEV_NORM = {
    # Prowler
    "critical": "CRITICAL", "high": "HIGH", "medium": "MEDIUM",
    "low": "LOW", "informational": "INFO",
    # Kubescape
    "Critical": "CRITICAL", "High": "HIGH", "Medium": "MEDIUM", "Low": "LOW",
    # Falco
    "EMERGENCY": "CRITICAL", "ALERT": "CRITICAL", "CRITICAL": "CRITICAL",
    "ERROR": "HIGH", "WARNING": "HIGH", "NOTICE": "MEDIUM",
    # Trivy
    "CRITICAL": "CRITICAL", "HIGH": "HIGH", "MEDIUM": "MEDIUM",
}

class BaseNormalizer(ABC):
    tool: str
    domain: str

    def _base(self, scan_id: str) -> dict:
        return {"id": str(uuid.uuid4()), "scan_id": scan_id,
                "tool": self.tool, "domain": self.domain,
                "first_seen": datetime.now(timezone.utc).isoformat(),
                "last_seen": datetime.now(timezone.utc).isoformat(),
                "status": "open"}

    def sev(self, raw_sev: str) -> str:
        return SEV_NORM.get(raw_sev, "INFO")

    @abstractmethod
    def normalize(self, raw_data, scan_id: str) -> List[dict]:
        ...

# normalizers/prowler.py
class ProwlerNormalizer(BaseNormalizer):
    tool = "prowler"; domain = "cspm"

    def normalize(self, raw_data, scan_id):
        findings = []
        items = raw_data if isinstance(raw_data, list) else [raw_data]
        for item in items:
            if item.get("Status") != "FAIL": continue  # skip PASS
            f = self._base(scan_id)
            f.update({
                "cloud":         item.get("Cloud", "aws").lower(),
                "severity":      self.sev(item.get("Severity", "")),
                "resource_type": item.get("ResourceType"),
                "resource_id":   item.get("ResourceId"),
                "resource_name": item.get("ResourceName"),
                "region":        item.get("Region"),
                "check_id":      item.get("CheckId"),
                "title":         item.get("CheckTitle"),
                "description":   item.get("Description"),
                "remediation":   item.get("Remediation", {}).get("Text"),
                "frameworks":    item.get("Compliance", []),
                "raw_output":    item,
                "tags":          ["cspm", item.get("Cloud", "aws").lower()]
            })
            findings.append(f)
        return findings

Community plugin contract
Any contributor can add a new scanner by providing:

1. plugins/toolname/plugin.yaml — manifest
2. plugins/toolname/normalizer.py — adapter
3. A Docker image (public or buildable)
4. A test with sample tool output

No core platform changes needed. Auto-discovered on startup.
Suggested license: Apache 2.0 — allows commercial use, modification, distribution. Compatible with all the OSS tools it wraps. Good for community contributions. Avoid GPL as it may conflict with some tool licenses.
